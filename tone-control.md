# トーン制御の段階設計

## 問題

LLM はデフォルトで「親しみやすい」トーンを志向する。
業務利用 (カウンセリング・社内 SaaS) では、馴れ馴れしさが信頼を損なう。

## 4 段階のトーン語彙

| 段階 | 想定読者 | キーワード |
|---|---|---|
| L1 親友 | プライベートなチャット | 「だよね」「〜してみて」 |
| L2 同僚 | カジュアル業務 | 「〜ですよ」「〜してみませんか」 |
| **L3 先輩・上司** | **業務向け推奨** | **「〜ですね」「〜なさってください」** |
| L4 カウンセラー | 専門性が必要 | 「〜と感じておられるようですね」「ご様子から」 |

## 実装パターン

### システムプロンプト断片 (L3)

```
信頼できる先輩や上司が、部下の話を受け止めて声をかける距離感で書いてください。
文体は「です・ます」基調の丁寧語。
馴れ馴れしい「〜だよ」「〜かな」「応援してるよ」等のため口は避け、
それでいて事務的・説教的にならず、相手を気遣う温かさを残してください。
```

### 効果

- GPT-4o / Claude Sonnet 共に、L3 指示で安定した敬語出力
- 温度 0.9 でも文体崩れが起きにくい
- 「親友」「友達」persona より instruction following が strict

## 失敗例

- 「親友のように」→ ため口混在
- 「丁寧に」→ 事務的・冷たい
- 「カウンセラーらしく」→ 距離が遠すぎ
